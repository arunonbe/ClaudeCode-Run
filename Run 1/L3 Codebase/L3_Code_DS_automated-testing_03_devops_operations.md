# DevOps / Operations View — DS_automated-testing

## Build System
- **Build tool:** Apache Maven (wrapper `mvnw` / `mvnw.cmd`)
- **Java version:** Java 8 (compiler source/target = 8)
- **Test framework:** TestNG 7.3.0 via Maven Surefire 3.0.0-M5
- **Module name:** `com.qa:UIAutomation:1.0`

## Test Execution
### Local / Manual
```
mvn clean test -Dsurefire.suiteXmlFiles=./testng.xml -Dtestng.dtd.http=true
```
Or via TestNG run configuration with VM option: `-Dtestng.dtd.http=true`

### Batch Scripts (legacy)
- `QADMTDeployRunTests.bat` and `QADMTDeployRunTests_v2.bat` — Windows batch wrappers for CI or scheduled execution on a Windows agent.

### Test Suites
| Suite File | Tests Included |
|------------|---------------|
| `testng.xml` | `OPPaymentLoginTest` (UI), `LegacyDBTest` (DB) |
| `DMTAPITest.xml` | DMT API tests (Config, System, User) |
| `DMTTestSuite.xml` | Larger DMT test suite |

## CI/CD
- **GitHub Actions:**
  - `.github/workflows/codeql-java.yml` — CodeQL static analysis for Java
  - `.github/codeql/codeql-config-java.yml` — CodeQL configuration
  - `.github/dependabot.yml` — Dependabot dependency alerts
- No automated test-execution pipeline; tests are run manually or via batch scripts.

## Dependencies (notable)
| Artifact | Version | Notes |
|----------|---------|-------|
| Selenium Java | 3.141.59 | Older release; WebDriverManager (4.3.1) handles driver download |
| TestNG | 7.3.0 | Stable |
| mssql-jdbc | 8.4.1.jre8 | SQL Server connectivity |
| mongo-java-driver | 3.12.8 + mongodb-driver 3.12.6 | Potential version conflict with mongodb-driver-core 4.2.2 |
| rest-assured | 4.3.3 | HTTP API testing |
| WebDriverManager | 4.3.1 | Auto-downloads browser drivers |

## Configuration and Credentials
- All environment URLs, DB hosts, usernames, and passwords are **hardcoded in Java source files** (see `Prerequisites.java`, `LegacyDBTest.java`).
- No `.properties` or environment-variable injection pattern in place.

## Observability
- Test results: TestNG HTML/XML reports generated by Surefire in `target/surefire-reports/`.
- `Log.java` and `General.java` — custom logging wrappers; output to stdout and a file via `MultiOutputStream`.
- `TestCaseWriter.java` — writes test case results to a file.
- No integration with any monitoring platform.

## Operational Risks
1. Hardcoded credentials will cause CI failures if passwords change and the source is not updated.
2. `mongodb-driver 3.12.6` and `mongodb-driver-core 4.2.2` are mixed versions — potential classpath conflict.
3. No test environment guard; a misconfigured suite file could target a production endpoint.
4. Selenium tests depend on browser versions matching the WebDriverManager-resolved driver; version drift can break runs silently.
5. Batch scripts (`QADMTDeployRunTests.bat`) have no error-handling or exit-code propagation.
