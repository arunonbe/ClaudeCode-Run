## Platform Generation (Gen-1/Gen-2/Gen-3)

**Generation: Gen-1**

Evidence:
- Apache Axis 1.4 (released 2006, EOL) as the SOAP transport framework.
- JAX-RPC 1.1 API ŌĆö superseded by JAX-WS 2.0 in Java EE 5 (2006) and then by Jakarta EE; no longer part of the Java standard.
- Spring Framework 4.3.9 (EOL Dec 2020).
- JUnit 3.8.1 ŌĆö predates JUnit 4 (2006).
- `1.0-SNAPSHOT` version with no release history visible.
- Spring XML-only configuration (no annotations, no Spring Boot, no auto-configuration).
- `System.out` / `System.err` for all logging ŌĆö pre-dates any structured logging adoption.
- Hard-coded Windows `d:/` drive path in Spring XML.
- WSDL stubs hand-edited / generated by Apache Axis WSDL2Java (auto-generation comment in test stubs: "This file was auto-generated from WSDL by the Apache Axis 1.4 Apr 22, 2006 (06:55:48 PDT) WSDL2Java emitter.").

This library represents the earliest generation of service-client patterns at Citi Prepaid, pre-dating both the micro-service era and the Onbe rebrand.

## Business Domain

**Authentication / Identity Federation** within the Prepaid Card domain.

The library exists at the intersection of:
- **Cardholder Experience**: enabling seamless SSO from Onbe's prepaid platform into the Cambridge FX Online portal (foreign exchange / multi-currency services for prepaid cardholders).
- **Partner Integration**: implementing the partner-defined SSO contract from Cambridge (Citibank's Cambridge FX partner).

The `com.citi.prepaid` package namespace confirms this originated within Citi's prepaid business unit.

## Integration Patterns

| Pattern | Implementation |
|---|---|
| **Service Locator** | `ServiceLocator.java` / `IServiceLocator.java` ŌĆö classic GoF Service Locator, deprecated in favour of DI |
| **Fa├¦ade** | `ICambridgeAuthService` / `CambridgeAuthServiceImpl` wraps SOAP complexity |
| **Stub / Proxy** | `BasicHttpBinding_ISSOServiceStub` ŌĆö Apache Axis-generated SOAP stub (RPC/Literal style) |
| **Spring XML DI** | `appContext-CambridgeAuthService.xml` wires dependencies declaratively |
| **SOAP 1.1 / BasicHttpBinding** | `_call.setSOAPVersion(SOAP11_CONSTANTS)` in stub line 120; endpoint name `BasicHttpBinding_ISSOService` |
| **HMAC-style shared-secret signature** | Hash-based digital signature over key fields ŌĆö though implemented with a plain hash (not HMAC); see `CambridgeAuthServiceHelper.java` |

There is **no message queue, event bus, REST, or gRPC** pattern. The integration is point-to-point synchronous SOAP.

## Strategic Status

| Dimension | Assessment |
|---|---|
| **Active maintenance** | Low ŌĆö SNAPSHOT version, no release tags, no changelog visible |
| **Security posture** | Poor ŌĆö EOL dependencies, MD5 risk, plaintext secrets, no TLS pinning |
| **Operational fit** | Poor for cloud/container ŌĆö hard-coded Windows path, global JVM proxy mutation, stdout-only logging |
| **Retirement risk** | High ŌĆö Cambridge FX partnership dependency; if that partnership ends or migrates APIs, this library has no value |
| **Migration urgency** | High ŌĆö EOL Spring 4.x, EOL Apache Axis 1.4, cryptographic weakness |
| **Strategic recommendation** | Replace with a modern REST/OAuth2 or SAML-based SSO integration if Cambridge exposes a modern API; or migrate to Spring WS / CXF with Spring Boot if SOAP is retained |

## Migration Blockers

| Blocker | Detail | Mitigation Path |
|---|---|---|
| **Apache Axis 1.4 SOAP stubs** | All integration code is Axis-specific (stub, type mappings, serializers). No JAX-WS or Spring-WS equivalent exists in repo | Regenerate stubs with Apache CXF or Spring-WS from a WSDL |
| **JAX-RPC 1.1 API** | `IServiceLocator extends javax.xml.rpc.Service` ŌĆö JAX-RPC removed from Java EE 6+ and not present in Jakarta EE | Replace `IServiceLocator` with JAX-WS `javax.xml.ws.Service` or Spring-WS `WebServiceTemplate` |
| **Hard-coded Windows path** | `d:/c-base/config/...` in Spring XML prevents containerisation | Externalise via environment variable or Spring Boot `application.yml` |
| **JVM-global proxy mutation** | `System.getProperties().put("http.proxyHost",...)` ŌĆö side-effect on shared JVM | Use Axis `_call.setProperty()` or dedicated `HttpClient` with per-call proxy |
| **Spring 4 XML-only configuration** | No `@Configuration` or `@SpringBootApplication`; requires Spring XML bootstrap | Migrate to Spring Boot 3.x with Java configuration |
| **MD5 algorithm default** | `algorithm` property defaults to MD5 in test; if inherited in production config, violates crypto standards | Enforce SHA-256 minimum in code; remove external configurability of algorithm |
| **No automated build CI** | No build workflow ŌĆö a rewrite cannot be safely validated in CI without adding one | Add a GitHub Actions `build.yml` for compile + test gate |
