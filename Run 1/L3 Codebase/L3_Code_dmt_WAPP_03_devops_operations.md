# DevOps & Operations Report — dmt_WAPP

## Build
- **No build pipeline exists.** The repository contains only pre-compiled Excel macro binaries (`.xlsm`).
- VBA source code is not exported or compiled through any automated process visible here.
- The `.gitignore` is a standard Visual Studio template (generated by gitignore.io), suggesting the original developer anticipated a compiled project but never materialised one.

## Deploy
- **Deployment is manual.** The production artefact (`Data Management Tool - Production.xlsm`) is placed on the RiskDB file share/server by hand.
- The link file (`DMT Production Copy Link.xlsm`) is distributed to users once and then self-updates by downloading from the server.
- No deployment scripts, Ansible playbooks, Terraform, or CI/CD pipeline artefacts are present.

## Configuration
- No environment configuration files exist in the repository.
- Connection strings, server names, and credentials are embedded in VBA macros inside the `.xlsm` binary — not visible or auditable from this repository.

## Observability
- No logging, monitoring, or alerting configuration.
- No application performance monitoring (APM) integration.
- Runtime errors surface only as Excel VBA runtime errors to the end user.
- Operational visibility is entirely dependent on RiskDB SQL Server auditing and Windows Event logs on the server.

## Infrastructure
- **RiskDB**: SQL Server instance on internal network (details not in this repo).
- **Distribution**: File share on RiskDB server (UNC path or mapped drive implied).
- **Client runtime**: Microsoft Excel (version not specified) on Windows desktops.
- No cloud infrastructure, containers, or orchestration involved.

## Operational Risks
| Risk | Severity | Notes |
|---|---|---|
| No CI/CD pipeline | Critical | Any developer can commit a new binary directly to the default branch; no gated review. |
| VBA code unauditable from Git | Critical | A malicious or faulty macro change is invisible in diff/PR review. |
| Manual deployment to RiskDB | High | No rollback mechanism; no deployment history beyond Git binary blobs. |
| No monitoring or alerting | High | Application failures are silent at the infrastructure level. |
| Excel version dependency | Medium | Macro compatibility depends on client Excel version; no version pinning or test matrix. |
| Single server dependency | Medium | RiskDB downtime = full DMT outage with no failover. |

## CI/CD Status
- **No CI/CD.** There is no `.github/workflows`, `.gitlab-ci.yml`, `Jenkinsfile`, or equivalent.
- Recommended path: integrate a macro export step (e.g. via `oletools`/`olevba`) into a GitHub Actions pipeline to produce a reviewable VBA diff on every commit.
