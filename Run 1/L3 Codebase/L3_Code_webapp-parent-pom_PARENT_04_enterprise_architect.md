## Integration Patterns

The parent POM establishes the following integration patterns for inheriting web apps:
- **Struts MVC with xDoclet**: Action-driven controller pattern where form beans and action mappings are generated by xDoclet from annotation-like Javadoc tags. This is a 2000s-era J2EE pattern with no dependency injection.
- **JSP view layer**: Struts Actions forward to JSPs; the JSPC profile pre-compiles JSPs for production WAR packaging.
- **Servlet container deployment**: WAR deployed to an external servlet container (not embedded).
- **EJB-era spec dependency**: `geronimo-j2ee_1.4_spec:1.1` confirms J2EE 1.4 era (circa 2003) design patterns.

## Position in the Broader Platform

This POM sits in the **governance layer** of the Gen-1 web application stack. It is the direct ancestor of all Struts-based web applications in the estate (ClientZone, CSA, Enrollment WAPP, Scheduler WAPP, etc.). Any change to this POM propagates to all inheriting modules.

In the three-generation platform model:
- Gen-1: This POM is the active parent for Struts web apps.
- Gen-2: Wirecard introduced `onbe-spring-boot-parent_PARENT` and `service-parent_PARENT` as Spring Boot replacements.
- Gen-3: NexPay uses `nexpay-parent` with Spring Boot 3.x and Java 21.

## Migration Blockers

1. **Struts-to-Spring MVC migration**: All child web apps must be rewritten to remove the Struts dependency before this parent can be retired. This is a significant effort given the xDoclet code-generation pattern.
2. **Java 6 bytecode**: Code compiled to Java 1.6 may rely on APIs or behaviors removed or changed in Java 9+. A full source-level migration and retesting is required.
3. **xDoclet dependency**: xDoclet 1.x is incompatible with Java 5+ annotations (`@Override`, generics). Migrating to Spring/Jakarta annotations requires rewriting all action classes.
4. **J2EE 1.4 spec dependency**: `geronimo-j2ee_1.4_spec` is a pre-Jakarta namespace artifact; migrating to Jakarta EE requires namespace changes across all source files.

## Strategic Status

**Critical legacy debt — active migration target.** This parent POM represents the oldest and most technically indebted layer in the Onbe estate. Immediate mitigations should include:
- WAF protection in front of all Struts-based apps (Req. 6.4 PCI DSS).
- Dependency upgrade plan for Struts → Spring MVC.
- Freezing the version at a release (removing SNAPSHOT) to prevent unintended cascading changes.
