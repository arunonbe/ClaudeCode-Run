# Solution Architect Report — windows-scripts

## API Surface

No formal API. The scripts invoke the eCount Core XML-RPC API through the J2COM bridge. The effective API surface of interest:

- `ECountCore.eMember.UpdateSecureProfile` — updates SSN and DOB for a cardholder.
- `ECountCore.eMember.InquirySecureProfile` — retrieves cardholder secure profile.
- `ECountService.DataEnvironment.Execute` / `ExecuteIgnorePK` — invokes arbitrary SQL Server stored procedures.
- Generic RPC via `RPC-EXEC.vbs` — any interface/method in eCount Core can be invoked by any operator.

## Security Posture

**Critically weak.** The scripts represent the most severe security and compliance exposures in the entire Gen-1/Gen-2 estate, with direct PII and financial data manipulation risks.

## Key Security Issues Detail

### Issue 1: Hardcoded SSN Example in Production Script
`vb-scripts/PROD/1099_ssn_update.vbs` line 18 contains a commented-out example with an apparent real SSN pattern: `"493025119"`. While commented out, this value should be examined to determine if it is a real SSN from a past correction operation that was inadvertently left in the code. If real, it constitutes a PCI DSS / GLBA violation as an SSN stored in version control.

### Issue 2: Unrestricted RPC Execution
`RPC-EXEC.vbs` is a generic XML-RPC dispatcher. Any user with access to the production server can execute any eCount Core interface method by invoking this script with `-a`, `-p`, and `-m` parameters. There is no authorization layer, no allowlist of permitted method invocations, and no audit log generated by the script itself.

### Issue 3: PK Violation Suppression
`data-exec-ignore-pk.vbs` intentionally suppresses primary key constraint violations (result code 12005) and treats them as success (RC=0). This pattern enables data to be silently skipped during bulk corrections, which can produce inconsistent database state without any alert.

## Technical Debt

- **Total rewrite required**: All VBScript operational scripts must be replaced before Windows VBScript removal.
- **Audit logging**: Every PII-modifying script must be rewritten with structured audit logging to a SIEM.
- **RBAC enforcement**: SSN/DOB modification must be gated by role-based access control, not server file-system access.
- **Test coverage**: Zero automated test coverage exists for any of these scripts.
- **Backup file cleanup**: Multiple `*_bkp07222021.vbs` files (dated July 22, 2021) are dead code in the repository. They must be removed to reduce confusion and ensure audit clarity.
- **Duplication**: Multiple variants of ACH control-total scripts for different banks (MB, Sunrise, Citi, Peoples) should be consolidated into a parameterized script.
