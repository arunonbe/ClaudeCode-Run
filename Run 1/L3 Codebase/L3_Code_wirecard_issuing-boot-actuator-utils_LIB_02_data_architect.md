# Data Architect Report — wirecard_issuing-boot-actuator-utils_LIB

## Data Models

The library operates on Spring Boot Actuator's `Health` domain model:

- **`Health`**: Composite object containing a `Status` and a `Map<String, Object>` of details.
- **`Status`**: Enumeration-like class with `UP`, `DOWN`, `OUT_OF_SERVICE`, `UNKNOWN` standard values.
- The serializer produces a JSON document with fields: `overall_status_ok` (boolean), `reply_host` (string), `cached_ts` (string), `status` (string), and nested health indicator objects.

No persistent data models. No database access. No entity classes.

## Sensitive Data

**Potentially sensitive — hostname disclosure.** The `CustomHealthSerializer` captures the server hostname via `InetAddress.getLocalHost().getCanonicalHostName()` at construction time and includes it in every health response as `reply_host`. In a cloud or container environment this may be:
- A Kubernetes pod name (revealing cluster naming conventions).
- An internal DNS hostname (revealing internal network topology).
- An EC2 instance ID or Azure VM name (revealing cloud infrastructure details).

If the health endpoint is accessible without authentication (as observed in the sg-bank-agent application.yml with `exposure.include: '*'`), this constitutes information disclosure relevant to PCI DSS Req. 12.3 (network security) and general attack surface reduction.

No PAN, CVV, SSN, DDA, or cardholder PII is processed by this library.

## Encryption Status

No data at rest or in transit is managed by this library. The health endpoint response is serialized JSON served over HTTP/HTTPS by the consuming Spring Boot application. TLS is the responsibility of the consuming service's configuration.

## Database Schemas

None. This is a pure serialization/aggregation utility library with no persistence layer.

## Data Flows

1. Spring Boot Actuator collects `Health` objects from all registered `HealthIndicator` beans.
2. `CustomHealthAggregator.aggregate()` combines them into a single composite `Health` object.
3. `CustomHealthSerializer.serialize()` converts the composite `Health` to the Wirecard-format JSON response.
4. The JSON is written to the HTTP response for the actuator health endpoint caller.

## Retention Concerns

None. No data is persisted by this library. Log entries generated by consuming services may include health status information in structured log output, but this is governed by the consuming service's logging configuration.

## PCI DSS Compliance

- **Req. 6.3**: The library is a reusable component. It should be versioned, released, and consumed via an artifact repository (GitHub Packages, Nexus) rather than as a source dependency.
- **Req. 7 / Req. 8**: The actuator health endpoint exposed by consuming services must be protected by authentication. This library does not enforce authentication; it is the responsibility of the consuming service's Spring Security configuration.
- The `reply_host` hostname disclosure via health endpoint, if unauthenticated, is a PCI DSS information disclosure concern (Req. 2.2 — secure configuration of system components).
